## üöÄ Deployment Instructions

### Step 1: Create GitHub Repository

1. Go to [GitHub](https://github.com) and sign in to your account
2. Click the "+" icon in the top-right corner
3. Select "New repository"
4. Name your repository (e.g., "portfolio")
5. Make it **Public**
6. Click "Create repository"

### Step 2: Upload Files

**Option A: Using GitHub Web Interface**
1. In your new repository, click "Add file" ‚Üí "Upload files"
2. Drag and drop all three files:
   - `index.html`
   - `styles.css`
   - `script.js`
3. Add a commit message (e.g., "Initial portfolio commit")
4. Click "Commit changes"

**Option B: Using Git Command Line**
```bash
# Navigate to your project folder
cd path/to/your/portfolio

# Initialize git repository
git init

# Add all files
git add .

# Commit files
git commit -m "Initial portfolio commit"

# Add remote repository
git remote add origin https://github.com/Adebayod84/portfolio.git

# Push to GitHub
git branch -M main
git push -u origin main
```

### Step 3: Enable GitHub Pages

1. In your repository, click "Settings" (gear icon)
2. Scroll down to "Pages" in the left sidebar
3. Under "Source", select "Deploy from a branch"
4. Under "Branch", select:
   - Branch: `main`
   - Folder: `/ (root)`
5. Click "Save"
6. Wait 1-2 minutes for deployment
7. Your site will be live at: `https://adebayod84.github.io/portfolio/`

### Step 4: Verify Deployment

1. Click the "Visit site" button in GitHub Pages settings
2. Your portfolio should be live and accessible!

**Development Process & Tools Explanation:**

This professional portfolio website was developed using a modern tech stack combining HTML5, CSS3, and vanilla JavaScript to create a responsive, visually distinctive web presence. The development process began with careful planning of the site architecture, focusing on semantic HTML structure for accessibility and SEO optimization. I utilized CSS Grid and Flexbox for responsive layouts that adapt seamlessly across desktop, tablet, and mobile devices. The design aesthetic was inspired by editorial and magazine layouts, featuring custom typography pairing (Crimson Pro serif for headlines and Darker Grotesque sans-serif for body text) sourced from Google Fonts. CSS custom properties (variables) were implemented for consistent theming and easy maintenance. Advanced CSS techniques include parallax scrolling effects, intersection observer animations for scroll-triggered reveals, smooth transitions, and hover state micro-interactions. JavaScript ES6+ was employed to enhance user experience through smooth scrolling navigation, dynamic section highlighting, mobile menu functionality, and performance optimizations like lazy loading. The website was version-controlled using Git and deployed to GitHub Pages, which provides free, reliable static site hosting with HTTPS security. The deployment process involved creating a public GitHub repository, pushing the codebase, and enabling GitHub Pages in the repository settings to automatically serve the site at a github.io domain. This approach demonstrates proficiency in frontend web development, responsive design principles, modern CSS/JavaScript features, and practical deployment workflows used in professional web development environments.
